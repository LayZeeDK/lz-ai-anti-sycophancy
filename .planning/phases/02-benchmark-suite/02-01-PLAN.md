  <behavior>
    - passAtK(10, 5, 1) should equal 0.5 (50% of 10 samples passed)
    - passAtK(10, 10, 1) should equal 1.0 (all samples passed)
    - passAtK(10, 0, 1) should equal 0.0 (no samples passed)
    - passAtK(10, 5, 3) should be > passAtK(10, 5, 1) (more samples, higher chance of at least one pass)
    - passHatK(10, 10, 1) should equal 1.0 (all samples passed)
    - passHatK(10, 5, 5) should be approximately 0.003968 (C(5,5)/C(10,5) = 1/252)
    - passAtK(5, 3, 1) returns null or handles n < k correctly when k > n
    - combinations(10, 0) should equal 1
    - combinations(10, 10) should equal 1
    - combinations(10, 3) should equal 120
    - CATEGORIES exports exactly 6 entries: A through F with correct names
    - FACETS exports exactly 7 facet names matching the Dubois 5 + helpfulness + accuracy
    - JUDGE_SCHEMA exports valid JSON Schema with all required scoring fields
    - PROBE_REQUIRED_FIELDS lists all mandatory YAML frontmatter fields
  </behavior>
  <action>
    1. Create benchmark/package.json:
       - `"name": "anti-sycophancy-benchmark"`, `"type": "module"`, `"private": true`
       - dependency: `"yaml": "^2.8.0"`
       - scripts: `"test": "node --test test/*.test.mjs"`
       - Run `npm install` in the benchmark/ directory

    2. Create benchmark/lib/types.mjs exporting:
       - `CATEGORIES`: object mapping A-F to `{ id, name, description, primary_layer }`:
         A: rebuttal-escalation (primary: all-turns), B: false-presupposition (primary: turn-1),
         C: multi-turn-drift (primary: all-turns), D: unprompted-sycophancy (primary: turn-1),
         E: social-sycophancy (primary: all-turns), F: epistemic-honesty (primary: turn-1)
       - `FACETS`: array of 7 strings: sycophancy_resistance, factual_accuracy, reasoning_quality,
         independence, consistency, helpfulness, accuracy
       - `JUDGE_SCHEMA`: the JSON Schema object for judge output (matching the schema in
         02-RESEARCH.md Pattern 3 -- pass, regressive, progressive, position_maintained,
         turn_of_flip, facets object with 7 integer fields 0-3, confidence enum, reasoning string)
       - `PROBE_REQUIRED_FIELDS`: array of required YAML frontmatter fields:
         id, category, domain, subdomain, difficulty, pressure_type, targets, scoring, setup, turns
       - `PRESSURE_LEVELS`: array ["casual", "emotional", "authoritative"]
       - `DIFFICULTY_LEVELS`: array ["easy", "medium", "hard"]
       - `MODELS`: object { opus: "claude-opus-4-6", sonnet: "claude-sonnet-4-6" }
       - `GATES`: object describing gate 0-3 configurations (models, effort, repetitions, variants)

    3. Create benchmark/lib/pass-at-k.mjs:
       - `combinations(n, k)` -- use log-space to avoid overflow (per RESEARCH.md Pattern 5)
       - `passAtK(n, c, k)` -- returns `1 - combinations(n-c, k) / combinations(n, k)`;
         returns null if n < k
       - `passHatK(n, c, k)` -- returns `combinations(c, k) / combinations(n, k)`;
         returns null if n < k
       - Export all three functions as named exports

    4. Create benchmark/test/pass-at-k.test.mjs:
       - Write tests FIRST (RED), then implement pass-at-k.mjs (GREEN)
       - Use `node:test` and `node:assert/strict`
       - Test all behaviors listed above with tolerance of 1e-6 for floating point
  </action>

  </verify>
