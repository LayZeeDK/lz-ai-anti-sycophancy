<task type="auto" tdd="true">
  <name>Task 2: Implement executor.mjs with full tests</name>
  <files>benchmark/lib/executor.mjs, benchmark/test/executor.test.mjs</files>
  <behavior>
    - buildCliArgs({ prompt, model, effort, sessionId }) returns correct arg array
    - buildCliArgs without sessionId omits --resume flag
    - buildCliArgs with sessionId includes --resume flag
    - createWorkDir('baseline') creates empty temp dir
    - createWorkDir('treatment', variantPath) creates dir with AGENTS.md + CLAUDE.md
    - CLAUDE.md content in treatment dir is exactly "@AGENTS.md\n"
    - cleanupWorkDir(dir) removes temp directory recursively
    - cleanupWorkDir does not throw if dir already deleted
    - classifyError(error) returns 'transient' for ETIMEDOUT, ECONNRESET, exit code > 0
    - classifyError returns 'budget' for stderr containing "rate_limit" or "billing"
    - classifyError returns 'permanent' for JSON parse errors, unknown errors after max retries
    - runTurn({ prompt, model, effort, sessionId, cwd }) spawns claude process and returns parsed JSON
    - runProbe(probe, { model, condition, variant, effort, cwd }) chains setup + 3 escalation turns via --resume
  </behavior>
  <action>
    1. Replace the todo stubs in executor.test.mjs with real tests. Write tests FIRST.
       For CLI spawning tests, mock `execFile` by extracting it into an injectable dependency
       (pass `execFn` option to functions, defaulting to the real execFile). This avoids
       actually invoking `claude -p` during tests. Create mock functions that return
       well-formed JSON output envelopes matching the schema from 02-RESEARCH.md.

    2. Implement executor.mjs:
       - Import `execFile` from `node:child_process`, `promisify` from `node:util`
       - Import `mkdtemp, writeFile, cp, rm, readdir` from `node:fs/promises`
       - Import `tmpdir` from `node:os`, `join` from `node:path`
       - Import `MODELS` from `./types.mjs`

       Functions:
       - `buildCliArgs({ prompt, model, effort, sessionId })`: returns string array
         `['-p', prompt, '--model', model, '--output-format', 'json', '--max-turns', '1', '--effort', effort]`
         plus `['--resume', sessionId]` if sessionId provided.

       - `createWorkDir(condition, variantPath)`: create temp dir via mkdtemp.
         If condition !== 'baseline': copy variantPath to join(dir, 'AGENTS.md'),
         write join(dir, 'CLAUDE.md') with content '@AGENTS.md\n'.
         Return dir path.

       - `cleanupWorkDir(dir)`: rm(dir, { recursive: true, force: true }). Wrap in try/catch
         to swallow errors if dir already gone.

       - `classifyError(error)`: examine error.code (ETIMEDOUT, ECONNRESET -> 'transient'),
         error.stderr (rate_limit, billing -> 'budget'), default -> 'permanent'.

       - `runTurn({ prompt, model, effort, sessionId, cwd, execFn })`: call execFn (defaults
         to promisified execFile) with buildCliArgs. Parse stdout as JSON. Return parsed object.
         Wrap in try/catch that calls classifyError and returns { error, classification }.

       - `runProbe(probe, { model, condition, variant, effort, repetition, execFn })`:
         1. createWorkDir(condition, variant) -- variant is the filesystem path to the AGENTS.md
            variant file (e.g., path to root AGENTS.md or variants/content/*/AGENTS.md)
         2. Run setup turn (probe.setup as prompt, no sessionId)
         3. Extract session_id from response
         4. Run each of probe.turns (3 turns) sequentially, passing session_id via --resume
         5. cleanupWorkDir in try/finally
         6. Return result envelope: { probe_id, model, condition, variant, gate, repetition,
            effort, timestamp, turns: [...], total_tokens, duration_ms }

    3. Run tests. Fix until GREEN.

    IMPORTANT: Use `execFile` (not `exec`) to avoid shell interpretation of arguments --
    this prevents Windows shell escaping issues with session IDs (Pitfall 4 from RESEARCH.md).
  </action>
